# BUG-001: Vercel 部署 API 返回空数据

**状态**: fixed
**日期**: 2026-06-13
**严重程度**: critical
**来源**: 代码审查

## 现象
Vercel 部署后，前端访问 API 始终返回空数据：
- `/api/articles?limit=5` → `{"total":0,"articles":[]}`
- `/api/search?q=游戏` → `{"total":0,"articles":[]}`
- `/api/stats` → `{"total":0}`

## 根因
三个关联问题导致 API 无法获取数据：

1. **Vercel 运行时无数据源**：`.vercelignore` 排除了 `reference/` 目录（markdown 源文件），`vercel.json` 设为 `echo prebuilt` 跳过构建，`lib/scanner.js` 兜底路径硬编码为 Windows 路径。API 冷启动时调用 `scanner.scanAll()` 找不到任何文件，返回空数组。

2. **搜索索引从未构建**：`api/search.js` 顶层只调用了 `scanner.scanAll()`，没有调用 `search.buildIndex(articles)`。`lib/search.js` 中 `miniSearch` 初始为 `null`，`search()` 方法第一行 `if (!miniSearch)` 直接返回 `[]`。

3. **预构建产物被闲置**：GitHub Actions 每 6 小时产出 `data/*.json` 并 push 到仓库，Vercel 部署包含这些文件，但 API 函数完全不读取它们。

## 修复
将 API 函数的数据源从"运行时扫描 markdown 文件"改为"读取 GitHub Actions 预构建的 JSON 文件"。保留 try-catch 回退到 scanner，兼容本地开发环境。

### 修改文件
- `api/articles.js` — 顶层 require `articles-list.json` + `articles-detail.json`；catch 回退 scanner + 实时渲染
- `api/search.js` — `fs.readFileSync` 读取 `search-index.json` → `MiniSearch.loadJSON` 重建索引；catch 回退 scanner+lib/search
- `api/stats.js` — 顶层 require `stats.json` 直接返回；catch 回退 scanner 动态计算
- `lib/scanner.js` — 兜底路径从 `C:\\Users\\jiehuiyang\\...` 改为 `path.join(__dirname, '..', 'reference')`（跨平台）+ console.warn 提示
