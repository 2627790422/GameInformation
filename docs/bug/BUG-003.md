# BUG-003: 生产环境文章不加载 — auth.js 缺失 + Vercel 路由拦截

**状态**: fixed
**日期**: 2026-06-28
**严重程度**: critical
**来源**: 用户报告（推送后网站无文章）

## 现象
访问 `game.yangjiehui.xyz`，页面空白，没有任何文章卡片渲染。列表页、详情页全部失效。

## 根因
两个独立问题叠加：

1. **[index.html:172](web-viewer/public/index.html#L172) 引用了 `<script src="/auth.js">`，但 `auth.js` 从未提交到仓库。**
   该文件和 Supabase CDN 引用 (`supabase-js`) 是本地调试时添加的，后续 `git add public/index.html` 把 index.html 变动（含 `<script src="/auth.js">`）提了上去，但 `auth.js` 本身遗漏。

2. **[vercel.json:9](web-viewer/vercel.json#L9) 的 catch-all 路由 `"/(.*)" → /index.html` 拦截了 `/auth.js`。**
   即便 `auth.js` 存在，vercel.json 只显式路由了 `/style.css` 和 `/app.js`，`/auth.js` 被 SPA fallback 规则重写为 index.html。

浏览器加载 `/auth.js` → 服务器返回 HTML → JS 解析报错 → 后续 `app.js` 不执行 → 所有文章逻辑停摆。

## 修复
1. 提交 `auth.js`（commit `84dbd5b`）
2. vercel.json 添加 `/auth.js` 路由（commit `81cdeeb`）

### 修改文件
- `public/auth.js` — 首次提交
- `vercel.json` — 新增 `{ "src": "/auth.js", "dest": "/auth.js" }` 路由规则
