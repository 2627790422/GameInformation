# BUG-002: 正文"处理时间"被错误提取为文章日期

**状态**: fixed
**日期**: 2026-06-15
**严重程度**: minor
**来源**: 用户报告

## 现象
文章"实测GLM-5.2，国产Coding模型的又一座新高峰"显示日期为 `2026-07-16`，实际应为 `2026-06-14`。该错误日期导致文章错误排到列表第一位。

## 根因
`lib/parser.js` 的 `extractDateFromBody()` 中正则 `/\*{0,2}(?:时间|日期)\*{0,2}[：:]\s*(.+)/` 过于宽泛，命中了正文末尾的 `*处理时间：2026-07-16*` 行（"处理时间"包含子串"时间"）。

body date 优先级高于 filename date，因此错误的 `2026-07-16` 覆盖了文件名 `26-06-14` 中正确的 `2026-06-14`。

## 修复

### 代码修复
- `lib/parser.js:50` — 正则添加负向后顾 `(?<!处理)`，排除"处理时间"模式

### 数据修复
- `data/articles-list.json` — GLM 文章日期 `2026-07-16` → `2026-06-14`
- `data/articles-detail.json` — 同上
- `data/stats.json` — 移除 2026-07 月份，合并到 2026-06

### 修改文件
- `lib/parser.js` — 正则加 `(?<!处理)` 排除处理时间
- `data/articles-list.json` — 修正单篇日期
- `data/articles-detail.json` — 修正单篇日期
- `data/stats.json` — 修正月度分布
