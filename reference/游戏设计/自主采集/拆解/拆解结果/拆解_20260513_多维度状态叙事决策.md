- **游戏**：Battlestar Galactica: Scattered Hopes
- **类型**：生存舰队管理肉鸽 + RTS战斗

## 四层拆解

### 行为层

#### 原行为（传统策略/肉鸽的默认行为模式）

- 只看单一状态维度（血量、资源量），刷到最优解后重复执行
- 遇到事件按固定优先级处理——能立刻解决就不拖延
- 战斗目标是"打赢"——清场、压制、以战力衡量胜负
- 死亡 = 失败，重开一局，上一局的经历只留下解锁货币

#### 新行为（BSG引入的行为变化）

- **不再只看血量**，改为同时追踪"阵营政治、医疗水平、维护程度"三个独立维度
- **不再只刷最优解**，改为在"立即处理（花钱）vs 延迟承担（吃惩罚）"之间反复权衡
- **不再清场制胜**，改为"撑到FTL跃迁准备完成即可逃离"
- **不再忽略内鬼**，改为在舰队管理中分配行动点调查渗透者——明知可能是"沉没成本"也要查
- **不再把死亡当失败**，改为将每局失败叙事化为"一支舰队的牺牲"，失败本身推进元进程
- **不再被动等事件**，改为主动利用状态维度触发针对性叙事——系统状态就是剧本

#### 行为本质变化

从"单维度资源最优化的确定性经营"变成"多维度状态博弈的不确定性生存管理"。玩家的核心行为从"算最优解"变成"在永远不完美的信息下做风险决策"。

---

### 系统层

#### 系统改变1：状态评估模型——从单轴到多面体

- **原本**：策略游戏的状态评估是一维的——血量/资源量/战力，玩家只需维持一个数值不归零
- **变为**：三个独立维度的多面体状态评估——阵营政治（内部分裂）、医疗水平（人员损耗）、维护程度（系统故障），每个维度独立恶化、交叉影响
- **设计含义**：状态不再是"扣血"的数值问题，而是"哪个维度先崩、连锁反应从哪个方向来"的定位问题

#### 系统改变2：叙事生成模型——从预设分支到程序化涌现

- **原本**：肉鸽的事件系统是设计师预设的 if-else 分支——"如果选择了X路线，触发事件Y"
- **变为**：事件不是预写的，而是由三个状态维度的数值交叉生成——"阵营政治30% + 医疗10% + 维护60%"自动计算出"维护危机中夹带政治丑闻"的叙事事件
- **设计含义**：程序化叙事让事件库不需要指数级膨胀——设计师只写"原子事件碎片"，系统根据状态自动化组合

#### 系统改变3：决策经济模型——从计算题到风险题

- **原本**：策略游戏的决策是有最优解的计算题——"花100金币解决问题 vs 承受80金币损失"，选前者
- **变为**：决策是风险题——"花100金币立刻解决 vs 承受未知的连锁惩罚（可能引发维度交叉恶化）"，前者是确定的代价，后者的损失范围不确定且可能雪球
- **设计含义**：用"不确定性"替代"代价比较"作为决策的核心张力——玩家永远不知道延迟处理的真实代价，这是恐惧驱动的选择

#### 系统改变4：胜利条件模型——从歼灭到存活

- **原本**：战术战斗的胜利条件是"消灭所有敌人"——战力比决定胜负
- **变为**：胜利条件是"在无限增援中存活到FTL倒计时结束"——存活时间决定胜负，清场无意义
- **设计含义**：将"打得过"的游戏变成"撑得住"的游戏，每次战斗都是消耗战，而非歼灭战

#### 系统改变5：元进程叙事模型——从"你死了重来"到"这支舰队牺牲了"

- **原本**：肉鸽的元进程是外挂的——死亡后弹出"解锁内容"界面，回到主菜单，叙事上不衔接
- **变为**：死亡被叙事化为"一支舰队牺牲了，另一支舰队接续逃亡"——元进程本身成为叙事的一部分
- **设计含义**：失败不再打断叙事连续性——这消除了肉鸽"重复感"的核心痛点

---

### 决策层

#### 短期决策（战斗中/单回合内）

1. **危机应对选择**：立即花钱止损 vs 承受负面效果——核心张力是"确定的代价 vs 不确定的后果"
2. **维度优先级分配**：当三个维度都在恶化时，先救哪个？——取决于"当前最紧迫的威胁"和"连锁反应路径预测"
3. **调查 vs 忽略渗透者**：分配有限的行动点调查内鬼——明知可能是沉没成本，但不查的后果更可怕
4. **战斗中的存活策略**：不是"怎么最快清场"而是"怎么让舰队活得最久"——把最肉的编队放前排拖时间，而非把输出最高的放前排清场

#### 长期决策（跨回合/build构筑）

- **资源分配的"维度投资"逻辑**：早期大量投入维护 → 中后期系统故障少，但可能政治/医疗崩溃；早期忽略维护 → 中后期系统崩盘引发多米诺效应。不再是"哪个更划算"的数学题，而是"你打算承受哪种崩盘方式"的风格选择
- **舰队构成的"抗性构建"**：不同起始舰队有不同的维度起始值——选择舰队不只是"选兵种"，而是选"你打算在哪个维度更有抗性"
- **元解锁的"叙事记忆"**：每次解锁不只是获取更强的单位/武器，而是"上一支舰队用牺牲换来的知识"——改变的不仅是战力，还有玩家对失败的叙事态度
- **风险偏好的培养**：系统训练玩家从"追求最优"转向"管理可接受范围内的风险"——这是从计算思维到战略思维的转变

### 情感层

#### 爽点1：不确定性中的掌控感

**为什么爽**：每次危机决策都没有最优解，但玩家每次做出判断后都能看到"至少我避免了更坏的情况"。这不是赢了爽，是"从更糟糕的命运中逃了出来"的侥幸感和成就感。

**典型场景**：三维护状态都在红线边缘，一个新危机弹出——你选择牺牲医疗换取维护，本回合撑过去了，医疗虽然崩了但至少船没炸。"差点就完蛋了"的后怕感是最强的。

#### 爽点2：连锁崩盘中的救场英雄感

**为什么爽**：当三个维度都恶化到临界点，在最后关头用一个关键决策连锁挽回局面——"政治谈判稳定了阵营"→"腾出资源修船"→"修好船加速FTL"——三连翻盘的快感远超单次最优解的执行。

**典型场景**：阵营政治崩盘前夕触发"谈判事件"，花大代价稳住 → 腾出的维护资源让FTL加速 → 在Cylon舰队到达前0.5秒跃迁成功。

#### 爽点3：内鬼揭发的叙事高潮

**为什么爽**：调查渗透者是"不可知恐惧→投入调查→碎片线索→真相揭露"的完整叙事弧线。等到最后揭发内鬼身份的瞬间，比单纯的"打赢一场战斗"更有故事感和成就感。

**典型场景**：怀疑了三个角色、调查了两个、花了两回合资源——第三个果然就是内鬼。在即将触发致命破坏前揭发，拯救全舰队。

#### 爽点4：以弱胜强的"逃出生天"

**为什么爽**：传统战斗是"我比你强所以我赢了"，存活导向战斗是"我比你弱但我活下来了"。FTL倒计时归零瞬间的跃迁逃逸，是纯粹的"绝望中的希望"。

**典型场景**：舰队残血、成员减半、系统濒临崩溃，但最后一秒成功跃迁。屏幕上的Cylon舰队在身后爆炸——不是因为你消灭了他们，而是因为你比他们多撑了一秒。

#### 爽点5：失败叙事的"意义感"

**为什么爽**：传统肉鸽的死亡只带来挫败感和刷币感。BSG将每次失败叙事化为"一支舰队的牺牲成为后续舰队的经验"，失败本身有叙事重量——不是"你输了"，而是"这支舰队完成了它的使命"。

**情绪类型**：高风险不确定性决策的恐惧-释放循环 + 叙事意义的赋予 + 以弱胜强的侥幸成就

## 玩法块公式

### 公式1：多维状态驱动事件 = 状态指标交叉计算 → 程序化事件组合

**原本**：单一状态触发预设事件（血量<30% → 触发"濒死事件A"）

**现在**：多维度状态交叉计算 → 程序化组合原子事件碎片生成叙事（阵营=30%+医疗=10%+维护=60% → "维护危机中夹带政治丑闻"）

**可迁移点**：任何需要状态追踪的系统都可以用"交叉计算"替代"条件触发"

### 公式2：危机决策 = 付出确定代价 vs 承受不确定后果

**原本**：决策是"代价A vs 代价B"的比较——哪个更划算选哪个

**现在**：决策是"确定的当前代价 vs 不确定的未来连锁反应"——确定性 vs 不确定性的风险偏好博弈

**可迁移点**：任何回合制决策系统都可以用"确定性代价 vs 不确定性后果"替代"数学最优解"

### 公式3：存活胜利条件 = 倒计时压力 + 无限压制

**原本**：胜利条件 = 消灭所有敌人 → 战力比决定胜负

**现在**：胜利条件 = 在无限增援的压制下存活到倒计时结束 → 存活时间决定胜负

**可迁移点**：在固定战力的游戏中引入"存活模式"——不要求赢，只要求活得久

### 公式4：嵌入式子系统 = 共用行动经济 + 不吃新UI

**原本**：新机制 = 独立小游戏/独立界面/独立资源

**现在**：新机制 = 与核心循环共用行动点和UI（内鬼调查从舰队管理的行动点中扣）

**可迁移点**：设计新系统时不新增UI层级，而是嵌入现有回合节奏

### 公式5：叙事性元进程 = 死亡叙事化 + 每局连续性

**原本**：肉鸽死亡 = 弹出解锁界面 + 回到主菜单

**现在**：肉鸽死亡 = "一支舰队牺牲了" → 元锁内容获得 → 下一支舰队接续故事

**可迁移点**：将"失败"叙事化为进程的一部分——不是"你输了"，而是"你完成了这局的故事"

## MCGG迁移方案

### 羁绊效果

#### 1. 多维度羁绊链（改编自"多维状态系统"）⭐⭐⭐⭐⭐

**效果描述**：
- 新增羁绊维度追踪：每个玩家的棋盘追踪 **"经济健康度"（金币/利息）、"羁绊凝聚力"（同羁绊棋子数量比例）、"血量压力"（剩余血量百分比）**
- 当某个维度低于阈值时，触发针对该玩家的棋盘事件
  - 经济健康度 < 30%："破产潮"——下回合商店棋子费用+1
  - 羁绊凝聚力 < 50%（包含太多散件）："内部分裂"——随机的非羁绊关键棋子本回合不发动技能
  - 血量压力 > 80%："背水一战"——所有棋子+10%伤害但每回合-5%最大血量

**适配度**：0.85（MCGG已有PvE模式和棋盘事件系统，隐藏维度在后台计算即可）

**建议羁绊名**：舰队意志 / 文明兴衰 / 局势感应

#### 2. 崩盘连锁羁绊（改编自"维度雪球效应"）⭐⭐⭐⭐

**效果描述**：
- 当一个羁绊完成度从高位跌落时，触发连锁惩罚
- 例如：从6刺客拆到4刺客 → "组织溃散"——剩余4个刺客棋子攻击力-15%，持续2回合
- 反向：从2刺客升到4刺客 → "凝聚成型"——新加入的刺客棋子首回合获得额外20%暴击率

**适配度**：0.75（需要改造——MCGG目前没有"羁绊降级惩罚"机制，但技术上可行）

#### 3. 潜伏者羁绊（改编自"Cylon渗透者"）⭐⭐⭐

**效果描述**：
- 2潜伏者：对局开始时，你的1个棋子被标记为"潜伏者"。潜伏者在第3回合战斗开始时揭示，揭示后获得+30%攻击力和技能冷却-2秒
- 4潜伏者：潜伏者揭示时，额外对敌方后排造成1.5秒眩晕

**适配度**：0.65（需要改造——MCGG需要实现延迟揭晓机制，但"以藏代打"的子类型在自走棋中有先例）

### 海克斯/Augment

#### 1. 危机管理 ⭐⭐⭐⭐⭐

- **级别**：金色
- **效果**：每回合开始时，你有两个选择：
  - A：支付2金币，本回合所有棋子获得15%伤害减免
  - B：不支付，本回合敌方全体棋子+10%攻击力
- **设计思路**：核心公式"确定代价 vs 不确定后果"的精简版——2金币的确定代价 vs 敌方全体buff的惩罚
- **适配度**：0.9（只增加一个选择弹窗，与现有海克斯系统完美兼容）

#### 2. 舰队跃迁 ⭐⭐⭐⭐

- **级别**：棱彩
- **效果**：当你的血量首次降至1时，触发"跃迁充能"——立即恢复至10血量，且下3回合每回合额外获得2金币。但第3回合结束时，如果你的血量仍≤10，直接淘汰
- **设计思路**：存活导向胜利条件的翻转——给你一次倒数计时翻盘的机会，但翻不了就GG
- **适配度**：0.80（需要在血量系统增加"跃迁充能"flag，技术上可行）

#### 3. 牺牲舰队 ⭐⭐⭐⭐

- **级别**：金色
- **效果**：当你本局被淘汰时，获得一个"舰队遗物"——下一局开始时额外获得2次免费刷新和1个随机4费棋子。遗物效果随连续失败叠加（上限3个遗物）
- **设计思路**："失败叙事化为进程"的元进程思想——连败不再是纯粹惩罚，而是"为下一局埋下伏笔"
- **适配度**：0.75（需要跨对局追踪"连续失败次数"，但MCGG已有海克斯解锁系统可扩展）

#### 4. 内鬼调查 ⭐⭐⭐

- **级别**：银色
- **效果**：选择一个敌方棋子进行"调查"。如果该棋子是该玩家的核心输出棋子（伤害占比>30%），则对其施加"暴露"——无法发动技能，持续1回合。如果不是，则你损失2金币
- **设计思路**："沉没成本风险"的推理嵌入——你需要通过观察敌方棋盘来"猜"谁是内鬼
- **适配度**：0.60（需要统计敌方棋子伤害占比并判断"核心输出"，且交互方式较新颖，需要UI改造）

### 棋子技能

#### 1. 舰队指挥官 ⭐⭐⭐⭐⭐

- **费用**：5费
- **羁绊**：战舰 / 谋略家
- **技能 - 危机抉择**：
  - 每场战斗开始前，在3个"舰队状态维"中选择2个激活：
    - 政治稳定：我方全体棋子+15%双抗
    - 医疗支援：我方血量最低棋子每秒回复5%最大血量
    - 工程维护：我方所有装备效果+20%
  - 未被选择的维度在当前战斗中变为敌方增益
- **设计思路**：核心机制"多维度交叉决策"的缩影——你必须在每场战斗前做权衡，总有牺牲
- **适配度**：0.88（多选项技能在MCGG已有先例，如塔姆/龙王的选择机制）

#### 2. FTL工程师 ⭐⭐⭐⭐

- **费用**：3费
- **羁绊**：工程师 / 时间操控
- **技能 - 跃迁充能**：
  - 被动：场上出现"FTL倒计时"（25秒循环）
  - 每当FTL倒计时归零，释放"跃迁脉冲"——将敌方所有棋子击退至棋盘底线，造成150/250/400魔法伤害
  - 但跃迁脉冲同时消耗我方1个随机棋子10%当前血量
- **设计思路**：存活导向+跃迁倒计时的双重表达——不是一次性技能，而是周期性节奏改变，每次都有代价
- **适配度**：0.82（周期性倒计时机制需要新增UI元素，但逻辑上可行）

#### 3. 渗透者 ⭐⭐⭐

- **费用**：4费
- **羁绊**：潜伏者 / 刺客
- **技能 - 伪装潜入**：
  - 第1-2回合：不攻击，伪装成敌方棋子外观，敌方看不到真实血量
  - 第3回合揭示：获得+50%攻击速度和+40%暴击伤害，持续到战斗结束
  - 如果在伪装期间被AOE命中3次，提前揭示但只获得一半增益
- **设计思路**：Cylon渗透者的"延迟揭晓"机制——玩家需要保护渗透者不被AOE打到才能获得完整收益
- **适配度**：0.68（伪装外观机制需要引擎改造，但延迟增益+条件触发在MCGG中可行）

## 设计启示

### 核心收获

1. **"不确定性决策"是MCGG可以深度挖掘的方向**——目前MCGG的决策多为"计算最优解"（DPS计算、羁绊完成度、装备分配），引入"确定代价 vs 不确定后果"的决策框架可以大幅提升策略深度

2. **"存活模式"作为PvE/PvPvE的设计锚点**——不要求玩家"打赢"，而是要求"撑住"，这种胜利条件更适合自走棋的随机性本质（弱势方也能有策略出口）

3. **"嵌入式子系统"原则**——设计新机制时不新增UI层级，而是融入现有回合节奏。海克斯选择弹窗已经为"多选项决策"留下了设计空间

4. **"叙事性元进程"降低连败挫败感**——将"失败"叙事化为有意义的进程，是解决自走棋"连败劝退"问题的创造性思路

### 不适合的方向

- **实时社交推理**与自走棋核心体验冲突较大——玩家预期是公平竞技，引入"谁是内鬼"会破坏棋盘策略的纯粹性
- **程序化叙事**需要大量事件库支持，MCGG不适合引入复杂的叙事系统
