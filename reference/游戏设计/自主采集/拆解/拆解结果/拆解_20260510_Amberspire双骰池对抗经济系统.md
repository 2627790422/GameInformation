## 四层拆解

### 行为层

**拿到点数估值系统后，玩家的行为发生了什么改变？**

| 维度 | 之前（精确配方） | 之后（点数估值） |
|------|-----------------|-----------------|
| 骰子结果感知 | 有用/没用（二元） | 高价值/中价值/低价值（连续） |
| 资源分配 | 查配方→匹配→用掉 | 评估资源对每个建筑的价值→最优分配 |
| 节奏控制 | 每轮固定数骰子 | 自主选择2-5颗骰子，主动控制风险 |
| 环境感知 | 天气是外部随机事件 | 天气是我投骰子"喂"出来的 |
| 失败的认知 | 不够资源=卡住 | 不够资源=进度慢≠失败 |

**行为本质变化**：
- 不再被动接受骰子结果，改为主动**估值后分配**（每颗骰子都在不同目标上有不同的"重量"）
- 不再把环境当外部干扰，改为**自我喂养的对抗力量**（投越多骰子→越快建造→同时也越强敌人）
- 从"回合制规划"变成"风险预算管理"（每次选择骰子数量都是一次风险决策）

---

### 系统层

**这个机制改了哪些底层规则？**

#### 1. 资源匹配模型：二元→连续估值

原本资源匹配是"符合/不符合"的二元判断：
- Furnace 需要 2金属+2砖 → 骰出金属=有用，骰出别的=白费
- 每颗骰子的结果只有 +0 或 +1 两种状态

改为点数估值：
- 每个建筑需要6点来建造
- 不同资源对同一建筑有 1-3 点不等的贡献
- 建造 Bazaar：Daylight=3点，Canvas=2点，Dye=1点
- 低价值资源虽推不动当前目标，但可能在别处是高价值

**本质改变**：资源从"同质化token"变成"有方向的向量"——不再问"够不够"，而是问"怎么配最值"。

#### 2. 风险生成模型：外部注入→自我喂养

原本：环境风险由系统随机生成（独立于玩家行为）

改为：玩家投骰子时，某些骰面标注"添加天气骰"
- 玩家主动选择投多少骰子 = 主动决定喂多少风险给环境
- 天气骰累积 → 每4回合集中爆发
- 大骰池 = 非线性增长的灾害风险（因为集中爆发比分散更难应对）

**本质改变**：风险不是"运气不好"，而是"你自己的贪婪"——玩家拥有风险的**所有权**。

#### 3. 时间结构：回合制→周期节拍

原本：每轮投骰→结算→下一轮（均匀节拍）

改为：多阶段周期结构
- 玩家阶段（复数）：选择骰子→投掷→获得资源→某些骰面累积天气骰
- 世界阶段（周期末尾1次）：所有累积的天气骰一次性投掷→地形生长/移动/侵蚀

**本质改变**：引入**延时反馈**——玩家当下的选择要在4回合后才"看到账单"，产生"欠债焦虑"和"集中爆发的戏剧性"。

### 决策层

**这个机制如何改变玩家的决策思路？**

#### 短期决策（每阶段即时选择）

1. **骰子数量决策**：投2颗还是5颗？
   - 少投=慢建造+少天气风险→安全但慢
   - 多投=快建造+多天气风险→激进但危险
   - 决策核心：估算"当前建筑的剩余点数" vs "天气骰池接近满的程度"

2. **资源分配决策**：这些资源投给哪个建筑？
   - 同一资源对不同建筑有不同价值
   - 决策核心：是推"高价值匹配"快速建好一个建筑，还是"低价值匹配"慢慢推多个建筑？
   - 存资源等待更好的建筑出现？还是现在就花掉？

3. **骰子选择决策**：哪些建筑放骰子？
   - 不仅要看产什么资源，还要看骰面是否带"添加天气骰"
   - 决策核心：追求产出效率 vs 控制风险输入

#### 长期决策（构筑方向改变）

1. **宽vs高构筑**：
   - 有估值系统后，建"多建筑=多骰子=多资源=多风险"的宽构筑 vs "少建筑=少骰子=少风险=精建"的高构筑
   - 宽构筑：资源多样化（每种资源都可能在别处高价值）→ 适应性强但风险高
   - 高构筑：专注少数建筑→能快速建成但应对力弱
   - 这改变了整个城市布局的策略逻辑

2. **建筑选择不再是中性的**：
   - 每个建筑不只是一张"产出卡"，它的"资源估值表"定义了资源的相对价值
   - 选择建筑A不选建筑B → 资源X从"高价值"变成"低价值"
   - 建筑组合形成**估值网络**，玩家需要构建互补的估值图谱

3. **风险节奏管理**：
   - 天气骰累积节奏决定何时"收缩"何时"扩张"
   - 在环境爆发前囤资源防身？还是在爆发后趁空隙猛冲？
   - 形成"呼吸式"的扩张-收缩节奏

### 情感层

**玩家用这个机制时的爽感是什么？**

#### 1. 精算成就感（智力快感）
为什么爽：每个资源对不同建筑有不同的"重量"，找到最优分配路径是对智力的直接奖励。不是"我有资源"的满足，而是"我比系统聪明"的快感。

**场景**：手里有 Daylight(3)、Canvas(2)、Dye(1)，面前有3个在建建筑。看出来 Daylight 投给 Furnace 最浪费（因为 Brick 只需要1点就能推一格）→ 重新分配后所有建筑都快建好了。这不是运气，是我算出来的。

#### 2. 风险喂养的张力（赌徒快感）
为什么爽：每次投骰子都同时推进两件事——建造进度和天气灾害。投越多越刺激（"再投一颗，应该没事吧？"），这是一种自驱动的紧张感。

**场景**：第3阶段，天气骰池已经快满了，但建筑还差2点。要不要再多投2颗骰子？投了可能在下个世界阶段被淹没，不投就慢一轮。这种"自己掐自己脖子"的张力是独特的。

#### 3. 集中爆发的戏剧性（一翻两瞪眼）
为什么爽：不是每回合都触发环境事件，而是攒4回合一次性爆。天气骰池越滚越大，世界阶段来临时"一次翻牌"带来强烈的戏剧高潮。

**场景**：攒了8颗天气骰，世界阶段开始——锈蚀花簇和粘性洪泛区同时爆发，L-system让它们相遇时"爆炸"成更大的图案，吞噬了你刚建好的半个城区。输是输了，但"输得精彩"。

#### 4. 非零和的满足感（没有烂骰）
为什么爽：即使最差的骰子面也有1点贡献。"至少推了进度"消除了完全浪费回合的挫败感。玩家不需要"每次都投出好结果"，只需要"每次都做对分配"。

**场景**：投了5颗骰子，4颗是低价值资源，只有1颗是高价值。但因为点数估值系统，这5颗加起来也有7点，刚好够建一个建筑。不是"运气好"，而是"分配好"。

#### 5. 叙事所有权（这是我的城市）
为什么爽：城市被侵蚀出了独特的形状——不是因为系统随机，而是因为"我上次贪婪多投了3颗骰子导致洪泛区从左边扩张"。城市的每一个疤痕都是我的决策后果，城市有我的故事。

**场景**：和朋友对比城市——"你看我这片被锈蚀花簇吃掉的区域，是因为我当时急着建Furnace投太多骰子。但你看我右边这片新扩展的城区，是在洪水过后重建的。"这不是AI生成的随机城市，这是我的城市。

**情绪类型**：计算支配感 + 风险张力 + 叙事主权

## 抽象玩法块公式

> 格式："原本XX，现在YY"

| # | 玩法块 | 公式 |
|---|--------|------|
| 1 | 资源匹配 | 原本资源与目标**精确匹配**（2金属+2砖=建成），现在资源对目标有**差异化估值**（1-3倍贡献，按资源-目标组合变化） |
| 2 | 风险来源 | 原本风险由**系统外部注入**（随机灾难、独立于玩家行为），现在风险由**玩家行为喂养**（玩家投骰子同时增加敌人力量） |
| 3 | 时间节奏 | 原本事件**分散逐轮触发**（每轮都可能发生），现在事件**延时集中爆发**（玩家行为持续累积，在固定节点一次性结算） |
| 4 | 进度模型 | 原本进度是**二元状态**（够/不够→建成/卡住），现在进度是**连续累积**（1-3点每次，6点建成，永远在推进） |
| 5 | 失败定义 | 原本失败是**游戏结束/重开**，现在失败是**局势变化**（城市被侵蚀=改变布局需求，不是Game Over） |

## MCGG迁移方案

### 羁绊效果

#### 方案1：点数估值羁绊「资源精算师」
```
(3) 资源精算师：每回合获得1颗"资源骰子"，投出1-6点。
    - 投出的点数可以分配给场上有羁绊激活的棋子：
      坦克棋子每点=+50生命值
      输出棋子每点=+3%攻击力
      辅助棋子每点=+5法力值
    - 不同棋子对不同点数的"估值"不同，
      玩家需根据当前局势选择最优分配

(5) 每回合获得2颗资源骰子
(7) 资源骰子点数上限+3（最高9点/颗）
```

**适配度**：0.75 — 点数分配逻辑需新UI，但数值层面MCGG完全支持

#### 方案2：自我喂养型羁绊「贪婪引擎」
```
(3) 贪婪引擎：你的棋子每击杀1个敌方单位，
    为敌方下一波棋子增加+2%攻击力。
    但每有1颗己方棋子存活，回合结束获得+1金币。

(5) 击杀给敌方+3%攻击力，存活给+2金币
(7) 击杀时己方全体永久+2%攻击力（无视敌方增益）
```

**适配度**：0.85 — 核心逻辑是"输出越强→敌人越强→需要更强输出"的正反馈循环，MCGG有现成的回合间状态传递机制

### 海克斯/Augment

#### 方案1：白银海克斯「风险投资」
```
每当你刷新商店，选择以下之一：
- 获得1金币（常规）
- 下次战斗敌方获得+5%伤害，但你获得2金币（风险选择）
此海克斯提供一种"自我喂养"的风险经济节奏
```

**适配度**：0.90 — 直接利用现有商店刷新操作，无需新增系统

#### 方案2：黄金海克斯「延时爆发」
```
每4个玩家回合，触发一次"世界阶段"：
- 投一颗特殊骰子（6面=6种全局效果）：
  ① 双方棋盘随机3格生成"锈蚀"（站在上面的棋子-30%攻速）
  ② 双方棋盘随机3格生成"洪泛"（站在上面的棋子每秒-2%生命值）
  ③ 己方全体+10%攻击力
  ④ 己方全体+10%护盾
  ⑤ 双方刷出4个免费英雄（限本回合）
  ⑥ 不触发
- "世界阶段"效果对所有存活的玩家生效
```

**适配度**：0.65 — 集中爆发的事件系统需要大量UI和逻辑支持，但核心"延时集中触发"的概念可简化迁移

#### 方案3：棱彩海克斯「非零和引擎」
```
你的棋子每次攻击/施法都产生1点"推进点数"
（无论命中与否、无论是否击杀）。
目标棋子获得30点推进点数后，该棋子永久+1星
（最高3星，每个棋子最多触发1次）。

推进点数可视化为棋子下方的进度条。
```

**适配度**：0.80 — 点数累积+进度条UI是MCGG可以实现的；核心迁移了"没有白费的操作"的设计哲学

### 棋子技能

#### 方案1：骰子型棋子「骰子大师」
```
费用：4费
技能（被动）：入场时投掷1颗专属骰子，
    决定本局战斗的模式：
    ① 伤害模式：普攻附带目标最大生命值5%的真实伤害
    ② 控制模式：普攻有30%概率眩晕目标1秒
    ③ 辅助模式：普攻为周围友军恢复2%生命值
    ④ 坦克模式：获得+50%生命值和+30双抗
    ⑤ 成长模式：每5秒永久+5%攻击力
    ⑥ 空过：本局无事发生

技能消耗：0法力（被动，入场即触发）
```

**适配度**：0.85 — 骰子决定模式是经典的自走棋棋子设计模式，MCGG完全支持

#### 方案2：自我喂养型棋子「贪婪领主」
```
费用：5费
技能（主动/50法力）：对所有敌方棋子施加"贪婪印记"（持续3秒）
    - 有印记的棋子受到伤害的20%转化为己方全体的治疗
    - 印记结束时，敌方棋子永久+5%攻击力
    （印记期间死亡不触发永久增益）

设计意图：
输出越高→治疗越多→但同时喂养敌方永久成长
玩家需要决定：是否使用贪婪领主？什么时候用？
```

**适配度**：0.70 — 需要实现"永久增益敌方"的回合间状态传递，技术上可行但需额外开发

#### 方案3：估值型棋子「资源商人」
```
费用：3费
技能（主动/80法力）：为场上所有友军棋子投一颗"估值骰子"
    - 投出点数分配给所有友军棋子
    - 1点=100护盾，2点=+30%攻速3秒，3点=+15%攻击力3秒
    4点=回复200生命值，5点=+1法力/秒3秒，6点=+40护甲+40魔抗3秒
    - 每个棋子只能获得1个点数（玩家分配哪个棋子得几点）
```

**适配度**：0.60 — 点数分配需要实时交互，自走棋的快节奏可能来不及操作；可简化为自动分配给"血量最低/攻击最高"等条件

## 设计方法论提取

Amberspire展示的**设计原子化（Design Atomization）**流程：
1. 分解每个系统元素 → 骰子面、资源类型、匹配规则
2. 识别问题根源 → "+0 or +1"的二元判断缺乏戏剧性
3. 引入新抽象 → 点数估值替代精确配方
4. 搭建反馈闭环 → 玩家行为喂养环境风险
5. 添加时间维度 → 周期结构制造集中爆发的戏剧性

**核心设计原则**：让"烂骰也有贡献"比"减少烂骰"更重要。
