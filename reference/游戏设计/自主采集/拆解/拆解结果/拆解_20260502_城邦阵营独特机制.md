## 四层拆解

### 行为层

**核心变化**：从"同一套开局最优解"→"城邦决定的差异化路径"

| 维度 | 无城邦 | 有城邦系统 |
|------|--------|-----------|
| 开局决策 | 所有人同一出发点 | 城邦选择创造3-4条不同路径 |
| 阵容路径 | 基于棋子池的通用最优解 | 城邦独有棋子/机制改变最优解 |
| 玩家行为 | 开局3回合基本固定操作 | 城邦特性引导差异化操作 |
| 信息获取 | 记住通用阵容表 | 了解每个城邦的独特机制 |

**关键行为转变**：城邦系统在每个维度上制造"分叉路"——同一局面下，不同城邦的玩家在做完全不同的事。这打破了自走棋"所有人都走同一条路"的结构性问题。

---

### 系统层

#### 系统1：城邦选择机制

| 阶段 | 设计 | 意图 |
|------|------|------|
| 对局开始 | 3-4个城邦随机展示 | 每次对局有新鲜感 |
| 选择阶段 | 玩家投票/随机分配 | 社交决策或运气博弈 |
| 城邦激活 | 独特机制/棋子/事件解锁 | 差异化体验 |

#### 系统2：城邦独特机制分类

| 类型 | 示例（TFT S16） | MCGG可参考方向 |
|------|----------------|---------------|
| 经济型 | 比尔吉沃特黑市（击杀获币） | 经济城邦：每击杀+2金币 |
| 养成型 | 绪塔尔太阳碎片（任务养成） | 养成城邦：棋子可积累经验值 |
| 条件型 | 恕瑞玛飞升（连胜/连败解锁） | 条件城邦：连胜3局激活特殊棋子 |
| 战斗型 | 德玛西亚残血反杀（低血量增益） | 战斗城邦：低于30%血时全队+攻击 |

#### 系统3：城邦与赛季轮换

| 维度 | 建议 |
|------|------|
| 城邦数量 | 每赛季3-4个核心城邦 |
| 轮换频率 | 每赛季轮换50%城邦 |
| 永久城邦 | 1-2个新手友好城邦保持不变 |
| 限定城邦 | 赛季限定，过期不候（增加稀缺性） |

### 决策层

#### 短期决策（城邦选择时）

- **城邦选择**：选哪个城邦？看自己擅长的流派，还是故意选不熟悉的？
- **城邦反制**：看到对手的城邦后，是否需要调整策略应对？
- **城邦信息利用**：如何最大化利用城邦独有机制？

#### 长期决策（赛季层面）

- **城邦专精**：专精1-2个城邦成为"城邦大师"
- **城邦适应**：每赛季学会3-4个城邦的核心玩法
- **城邦选择博弈**：在投票选择城邦时，如何影响结果？

#### 决策复杂度评估

| 维度 | 无城邦 | 有城邦 |
|------|--------|--------|
| 开局决策点 | 1个（首轮选秀） | 2个（城邦选择+首轮选秀） |
| 策略路径数 | 有限（阵容表决定） | 城邦×阵容=指数级增长 |
| 重复可玩性 | 中（阵容会玩腻） | 高（不同城邦=不同游戏） |

### 情感层

#### 爽点分析

| 爽点 | 为什么爽 |
|------|---------|
| **新鲜感** | 每次开局随机城邦 = "这次会抽到什么？"的期待 |
| **归属感** | "我是恕瑞玛玩家"——城邦成为身份标签 |
| **掌控感** | 精通某个城邦后，在这个城邦中"这就是我的主场" |
| **叙事感** | 城邦有故事背景，战斗不只是数值，而是"恕瑞玛vs德玛西亚" |

#### 情绪类型

- **期待 (Anticipation)**：开局等待城邦揭晓
- **归属 (Belonging)**：城邦成为玩家社交身份
- **怀旧 (Nostalgia)**：喜爱的城邦轮换回归时的感动
- **探索 (Exploration)**：新赛季新城的未知期待

#### 潜在风险

- **平衡地狱**：城邦越多，平衡越难（14个城邦=14倍的平衡工作）
- **被迫选择**：如果某个城邦过强，玩家被迫选它，失去"选择"的意义
- **学习成本**：新玩家面对3-4个城邦的独特机制可能不知所措

## MCGG迁移方案

### 城邦设计（3个示例）

#### 城邦1：暗影之渊（Shadow Abyss）

| 特性 | 内容 |
|------|------|
| **主题** | 暗影/牺牲/复活 |
| **核心机制** | 棋子死亡后以50%属性复活一次 |
| **独有棋子** | 暗影领主（5费）：亡语后召唤2个暗影仆从 |
| **独有海克斯** | 暗影契约：献祭一个棋子，获得2倍金币 |
| **适合流派** | 牺牲流、亡语流、经济运营流 |

#### 城邦2：元素之庭（Elemental Court）

| 特性 | 内容 |
|------|------|
| **主题** | 元素/转换/适应 |
| **核心机制** | 每3回合，元素属性轮换（火→水→风→土），影响所有棋子 |
| **独有棋子** | 元素使（3费）：根据当前元素改变技能效果 |
| **独有海克斯** | 元素亲和：本局固定元素不再轮换 |
| **适合流派** | 适应流、多核流 |

#### 城邦3：时间裂隙（Time Rift）

| 特性 | 内容 |
|------|------|
| **主题** | 时间/重来/预知 |
| **核心机制** | 每局可"回溯"一次（撤销上回合操作） |
| **独有棋子** | 时光行者（4费）：技能使目标攻击速度±50% |
| **独有海克斯** | 预知未来：查看下回合对手阵容 |
| **适合流派** | 高风险高回报、信息博弈流 |

### 落地建议

```text
阶段1：3个基础城邦（暗影/元素/时间）为赛季主题
阶段2：每赛季新增1-2个城邦，轮换旧城邦
阶段3：城邦专属赛事——"暗影之渊锦标赛"
```

## 设计总结

| 维度 | 核心洞察 |
|------|---------|
| **行为层** | 城邦系统在开局制造"分叉路"，打破自走棋"所有人走同一条路"的结构性问题 |
| **系统层** | 城邦 = 赛季主题容器，将独特机制+棋子+海克斯打包为完整体验 |
| **决策层** | 城邦选择成为每局第一个重大决策，影响后续所有策略路径 |
| **情感层** | 城邦提供"主场感"和"归属感"，让玩家不只是上分，而是"为自己城邦而战" |
