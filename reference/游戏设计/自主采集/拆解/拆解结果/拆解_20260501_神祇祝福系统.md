# 拆解结果：神祇祝福系统（Realm of the Gods）

## 基本信息
- **来源**：../研究/研究结果/研究_20260501_神祇祝福系统.md
- **拆解时间**：2026-05-01 10:21 UTC
- **来源游戏**：TFT Set 17
- **适配度**：0.88

---

## 四层拆解

### 第一层：行为层

**拿到祝福系统后，玩家的行为变化**：

| 原有行为（Carousel模式） | 变化后行为（神祇祝福模式） |
|--------------------------|---------------------------|
| 抢装备/抢棋子，拼手速和反应 | 点选祝福，纯策略判断 |
| 每次选秀独立决策，互不关联 | 递进式决策链，每次选择影响后续 |
| 只看"当前最强"的选择逻辑 | 看"当前+未来"的规划逻辑 |
| 落后方靠选秀顺位补偿 | 落后方靠Pengu机制+祝福选择共同补偿 |
| 装备/棋子是战利品 | 祝福是"战略方向牌" |
| 开场选秀一次性决策 | 全游戏分阶段多次决策（1-1/2-4/3-4/4-7） |

**行为本质变化**：从"抢东西吃"变成"选路线走"。核心是把原本的**反应操作竞争**改为**策略规划竞争**，每一轮祝福选择都在回答一个问题：你这一局想走什么路？

---

### 第二层：系统层

**底层规则被改了什么？**

#### 1. 核心循环机制替换：操作竞争 → 策略选择
- **原本**：Carousel阶段，所有玩家在公共区域实时抢一件装备/棋子，手速快者得、反应慢者失
- **变成**：祝福选择阶段，玩家独立点选祝福（3选1），无时间压力，纯策略判断
- **系统意义**：将"物理操作"从自走棋核心循环中剥离，使自走棋回归"策略游戏"而非"动作游戏"

#### 2. 积累系统引入：离散决策 → 递进决策链
- **原本**：每次选秀独立存在，选完就结束、下次重来、互不关联
- **变成**：祝福选择 → 好感度累积 → 影响最终Boon质量，前期选择有后期"利息"
- **系统意义**：引入了"决策复利"机制——玩家的早期选择不仅影响当下，还通过好感度在终局结算回报，建立了跨游戏阶段的因果链

#### 3. 落后补偿机制重构：单层补偿 → 双层补偿
- **原本**：落后补偿只有"选秀优先顺位"一条路
- **变成**：祝福选择本身有Pengu补偿（低血量额外选择/强化）+ 不同祝福类型适配不同局面（领先选经济扩大优势、落后选战力止损、胶着选容错保障）
- **系统意义**：落后补偿从"给你先选"的机械补偿，变为"根据局面选策略"的有机补偿

#### 4. 经济系统与祝福的交互
- **原本**：金币只用于升级/刷新/D牌
- **变成**：终局Boon阶段可以用金币重roll祝福，增加了金币→祝福的经济出口
- **系统意义**：为金币系统增加了新消费场景，丰富了经济决策维度

### 第三层：决策层

#### 短期决策（战斗中/当回合）

| 决策点 | 决策内容 | 思考逻辑 |
|--------|---------|---------|
| 开局1-1选神祇 | 二选一，选哪位神祇供奉 | 快速判断：我想要什么风格的祝福？（经济型/战力型/容错型） |
| 前期2-4小祝福 | 3选1，选择具体小祝福 | 当前阵容缺什么？这个祝福对未来3回合有什么帮助？ |
| 中期3-4中祝福 | 3选1，选择具体中祝福 | 累积好感度是否要延续？还是改方向？当前局面需要啥？ |
| 后期4-7大Boon | 获得Boon + 可重roll | 现在最缺什么？要不要花金币重roll？ |

**决策链条画像**：1-1选方向 → 2-4积累 → 3-4强化或转向 → 4-7收获，每个决策都在为下一个决策铺路。

#### 长期决策（build构筑）

神祇祝福系统改变后续构筑选择的方式：

- **经济型神祇（如Ahri）** → 走等级压制路线，快速上人口 → 后续选择高费棋子为核心的阵容
- **战力型神祇** → 走质量压制路线，前期加强战力 → 后续选择偏前期发力的阵容/装备
- **容错型神祇（如Soraka）** → 走稳扎稳打路线，容错率提升 → 后续可以承受更高的风险操作（贪经济、卖血D牌）
- **特殊型神祇（如Yasuo）** → 走Hex强化路线，围绕特殊机制构建 → 后续选择配合Hex的棋子/羁绊

**关键特征**：祝福选择不是"给装备"式的战术加强，而是"定方向"式的战略选择。它影响了玩家整局的资源分配优先级（花钱升级还是D牌？追求连胜还是贪经济？）

### 第四层：情感层

#### 爽点分析

| # | 爽点 | 为什么爽 | 触发场景 |
|---|------|---------|---------|
| 1 | **规划的快感** | 不是被动应对，而是主动设计自己的游玩路线。从开局选神祇那一刻起，玩家就在构建"我的局" | 开局选神祇、每次祝福选择 |
| 2 | **累积的满足感** | 前期埋下的种子在终局开花——好感度累积到Boon结算的那一刻，"我早就计划好了"的成就感 | 4-7 Boon触发 |
| 3 | **独特性认同** | 每位玩家选了不同神祇、走了不同路线，"我的祝福配置"是一种身份标识，不是"大家都抢同一件装备" | 全程，看到其他玩家的不同选择 |
| 4 | **逆转的希望感** | Pengu补偿 + 祝福策略选择 = 即使落后也有"我还能翻"的信念，不是"坐等老八" | 低血量获得补偿祝福时 |
| 5 | **重roll的赌博刺激** | 花金币重roll Boon = 小赌一把，花1金翻倍收益的瞬间刺激 | 终局重roll出好结果 |
| 6 | **操作解放的舒爽** | 不再需要拼手速、抢装备、卡位——静下心来做策略选择，移动端体验大幅提升 | 每个祝福选择节点 |

**情绪类型**：策略掌控感 + 长期积累满足 + 身份认同 + 逆转希望

**情绪弧线设计**：
- 开局 → 期待（会遇到哪位神祇？）
- 中盘 → 规划满足（我选对了！）
- 终局 → 高潮释放（Boon结算！）
- 落后时 → 希望维持（Pengu救我！）

## 玩法块公式

### 公式抽象

| 玩法块 | 公式描述 |
|--------|---------|
| **核心替换** | 原本Carousel是"实时抢东西"的**操作竞争**，现在变成"回合点选祝福"的**策略选择** |
| **决策链化** | 原本每次选秀**独立离散**，现在祝福选择**递进累积**，前期决策有后期利息 |
| **补偿优化** | 原本落后补偿只有"你先选"的**单一维度**，现在变成"根据局面选策略"的**多维度补偿** |
| **经济扩展** | 原本金币只用于升级/刷新/D牌，现在增加了金币→祝福重roll的**新经济出口** |
| **方向锚定** | 原本装备选完就定型，现在祝福选择给出**整局战略方向**，影响后续所有构筑决策 |

### 设计模式提炼

**"分阶段递进选择 + 跨阶段累积回报"模式**

```
Stage 1（开局）：粗粒度方向选择（神祇二选一）
     ↓ 影响
Stage 2（前期）：中粒度策略选择（小祝福三选一）
     ↓ 累积好感度
Stage 3（中期）：中粒度强化选择（中祝福三选一）
     ↓ 好感度结算
Stage 4（终局）：全局奖励释放（Boon + 可重roll）
     ↓ 奖励反馈
终局胜负
```

## MCGG迁移方案

### 羁绊效果

**方案：祝福羁绊（GoGo Blessing Synergy）**

设计一个特殊羁绊 `祝福使徒`，启用棋手祝福系统：

| 羁绊等级 | 效果 |
|---------|------|
| (1) | 局内每5回合触发一次祝福选择（3选1），选择持续3回合的全局buff |
| (3) | 祝福选择提升为4选1，且每次选择额外积累1层好感度 |
| (5) | 终局（第25回合）触发Boon，基于累积好感度获得强力全局效果 |

**设计理由**：将祝福系统作为羁绊效果引入，让玩家主动选择是否"走祝福路线"，不强制所有对局使用。

### 海克斯/Augment

**方案A：神祇的恩赐（银/金/彩海克斯）**

| 等级 | 效果 | 适配度评估 |
|------|------|-----------|
| 🥈 银色 | 在下一回合获得一次额外祝福选择（2选1），持续3回合 | 0.85 |
| 🥇 金色 | 每次自然祝福选择从2选1变为3选1，持续整局 | 0.80 |
| 🏆 彩色 | 获得一个随机大Boon（全局永久buff），且每5回合可花1金币重roll | 0.78 |

**理由**：海克斯是MCGG的特性位，天然适合承载祝福类的"一次性规则修改"效果。

**方案B：指挥官祝福（棋手专属海克斯）**

| 棋手 | 祝福效果 | 适配度 |
|------|---------|--------|
| 任一棋手 | 该棋手携带"指挥官祝福"海克斯后，每轮战斗胜利积累祝福值，祝福值满时触发一次全局buff | 0.88 |

**理由**：直接绑定棋手指挥官系统，复用现有的棋手Skill配置体系。

### 棋子技能

**方案A：祝福祭司（5费棋子的祝福光环）**

- **棋子名**：祝福祭司
- **费用**：5费
- **技能**：入场时给所有友军施加一个祝福buff，每轮战斗后buff层数+1。3层时触发Boon：所有友军获得持续到战斗结束的全局强化（攻击力+15%、法强+15、血量回复+100/秒）
- **设计思路**：将祝福系统压缩为一个棋子的技能效果，低开发成本高验证速度
- **适配度**：0.85

**方案B：神谕者（祝福重roll棋子）**

- **棋子名**：神谕者
- **费用**：3费
- **技能**：每场战斗开始前，随机获得一个本轮生效的小祝福（攻击力、防御、攻速、回蓝等4种中随机1种）。可以花1金币重roll祝福类型。
- **设计思路**：保留祝福的核心体验（选择+重roll），压缩为一个3费棋子的被动技能
- **适配度**：0.90

## 设计洞察

### 这个机制的"魂"是什么？

**核心矛盾**：自走棋的Carousel一直是操作型机制（抢装备），而自走棋本质是策略游戏。神祇祝福解决的就是这个错位——**把操作竞争拉回策略竞争**。

**迁移到MCGG的核心价值**：
1. **移动端友好**：MCGG是手游，"选"永远比"抢"更友好
2. **差异化定位**：把MCGG与TFT的Carousel差异化，建立"MCGG是策略选择型的自走棋"的心智
3. **策略深度**：分阶段递进选择比单次选秀深得多

### 一句话总结

> 把"抢东西"变成"选路线"，让每一次选择都有长期意义。
